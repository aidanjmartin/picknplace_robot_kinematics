# 6-Bar Pick-and-Place Robot Kinematics

MATLAB simulation of a 6-bar linkage with a prismatic joint acting as an
industrial pick-and-place robot. Built for ENGR 3590 (Kinematics and
Dynamics of Machinery) at Middle Tennessee State University.

The robot lifts boxes from a high conveyor and deposits them onto a lower
drop-off zone. Trajectories are generated by inverse kinematics, with a
brute-force workspace scanner choosing elbow positions that keep the
linkage inside its physical reach limits.

## Files

| File | Purpose |
| --- | --- |
| `main.m` | Full pick-and-place animation driver. Runs the workspace scanner, builds the multi-phase trajectory, and renders each frame. |
| `kinematic_viz.m` | Renders a single static pose of the linkage for use in figures. |
| `solve_IK.m` | Inverse kinematics: given an end-effector target `(X, Y)`, returns the four joint angles via the Law of Cosines. |
| `get_coordinates.m` | Forward kinematics: given joint angles and slider extension, returns the joint coordinates needed to plot the linkage. |
| `solve_positions.m` | Position-loop solver: returns the passive coupler/rocker angles `(t3, t4)` from the driven crank angles `(t2, t5)`. |
| `solve_velocities.m` | Velocity-loop solver: returns `(t3_dot, t4_dot)` from the driven crank velocities by solving `A*x = B`. |
| `ENGR3590_FinalProject.pdf` | Project report covering the kinematic analysis, design iterations, and conclusions. |

## Linkage parameters

| Link | Length (in) | Description |
| --- | --- | --- |
| `r1` | 4 | Ground link between the two base pivots |
| `r2` | 5 | Left crank |
| `r3` | 6 | Left coupler |
| `r4` | 6 | Right rocker |
| `r5` | 5 | Right crank |
| `r6` | variable | Prismatic slider extension along the coupler axis |

## Running

Open the project folder in MATLAB and run either:

```matlab
main            % full pick-and-place animation
kinematic_viz   % static presentation pose
```

An optional `background.png` can be placed in the project folder; if
present, `main.m` renders it behind the linkage.

## Approach

The trajectory is driven by **inverse kinematics** rather than forward
kinematics so that target coordinates (conveyor surfaces, drop points)
are inputs rather than outputs — this avoids iterative angle-guessing
and guarantees the gripper lands on the correct surface.

Before the animation runs, a **2D workspace scanner** sweeps candidate
elbow positions for each pick and drop target. For each candidate it
checks the triangle-inequality bounds on both arms and computes the
slider extension required to reach the target along the coupler axis
(`r6 = ΔY / sin(t3)`). The candidate with the smallest tip error is
selected. This eliminates singularities and prevents `acos()` from
returning complex numbers when a target sits outside the reachable
workspace.

See the report for full derivations and the design history.
